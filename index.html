welcome!

positioning in css can be a pain to wrap your head around! i know it befuddled me at first. so, here is a concise but effective cheat sheet to help you master element positioning in css!

❀

using the position property

actually using this property is quite easy! however, they do differ slightly. don't worry i'll show you how it's done.

here is a table that displays the five positions an element can take. read on to check out some examples!

Property Use
static this is the default state of an element. an element with this position is always positioned according to the normal flow of the page.
relative an element with this position is positioned relative to its normal position in the document flow. setting top, right, bottom, and left properties will cause the element to be adjusted away from its normal position. other content will not be adjusted to fit.
fixed an element with this position is positioned relative to the viewport — this means it will remain in the same place even during scrolling. it does not leave a gap where it would normally be located.
absolute an element with this position is positioned relative to the nearest positioned ancestor (with a position other than static). however, if there are no such positioned ancestors, it uses the document body, and moves along with page scrolling. these are removed from the normal document flow and can overlap elements.
sticky an element with this position toggles between relative and fixed positioning, depending on the scroll position. a sticky element is positioned relative until a certain scroll point is reached, in which it "sticks" in that place.

❀

static

static is the default positioning for elements on a webpage, however it is helpful to know how to set an element's position to static in some cases — for example, when overriding previously stated styles.

this text is static, so is the block holding the code, and the table above!

div {
  position: static;
}

relative

relative positioning requires the top, right, bottom, and left properties to be set, this causes the element to shift the distance set away from its normal position.

if we take the squares below for example, you can see the pink square has been positioned 50px from the left (to the right) and 20px from the top (down) away from it's original space (refer to the static example to view its original position).

.square-pink {
  position: relative;
  left: 50px;
  top: 20px;
}

fixed

fixed positioning will remain in the same place on the page even when scrolling. top, right, bottom, and left are used to set the final location of an element.

it is different from the absolute positioning, as it does not respect any parent with the relative positioning, and will instead respect only the viewport. in the example below only, please take the white box as a visual representation of the viewport (the corners have been adjusted square to aid in this representation). we can see that the pink square is 0px from the top and the left.

.square-pink {
  position: fixed;
  top: 0;
  left: 0;
}

absolute

elements positioned with absolute are removed from the flow of the document and can overlap elements. the properties top, right, bottom, and left are used to offset the element in relation to the closest ancestor.

in the example below, you can see that the pink square has been positioned 20px from the left (to the right) and 10px from the top (down), relative to its parent (the green square, re: the html).

html
<div class="square-green">
  <div class="square-pink"></div>
</div>
<div class="square-blue"></div>

css
.square-pink {
  position: absolute;
  margin: 0;
  top: 10px;
  left: 20px;
}

.square-green {
  position: relative;
}

sticky

with this positioning, an element will act as though it has been positioned with the relative property until it reaches the point of the page it has been assigned, where it will then "stick" to that position.

in the example below, you can see that the green square "sticks" to the top of the page when you scroll because it is set to stick 0px from the top when scrolling.

.square-green {
  position: sticky;
  top: 0;
}
