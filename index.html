using the position property

actually using this property is quite easy! however, they do differ slightly. don't worry i'll show you how it's done.

here is a table that displays the five positions an element can take. read on to check out some examples!

Property Use
static this is the default state of an element. an element with this position is always positioned according to the normal flow of the page.
relative an element with this position is positioned relative to its normal position in the document flow. setting top, right, bottom, and left properties will cause the element to be adjusted away from its normal position. other content will not be adjusted to fit.
fixed an element with this position is positioned relative to the viewport — this means it will remain in the same place even during scrolling. it does not leave a gap where it would normally be located.
absolute an element with this position is positioned relative to the nearest positioned ancestor (with a position other than static). however, if there are no such positioned ancestors, it uses the document body, and moves along with page scrolling. these are removed from the normal document flow and can overlap elements.
sticky an element with this position toggles between relative and fixed positioning, depending on the scroll position. a sticky element is positioned relative until a certain scroll point is reached, in which it "sticks" in that place.
